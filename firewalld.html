Notes on Firewalld

Firewalld is the default for RHEL, and is provided by two packages:

  • firewalld
  • firewall-config

Firewalld has two modes possible:

  • Runtime changes (don't survive at reboot or reload of the firewall).
  • Permanent changes.
Firewalld's main command is firewall-cmd.

Some commands

  • Firewalld automatically creates some zones with different rules, to make easier apply changes only to certain situations (eg internal network, dmz...). To list all the zones firewall-cmd --get-zones.
  • The default working zone it's the public one. To get the default zones, firewall-cmd --get-default-zones.
  • To view all the rules actually running on my system, firewall-cmd --list-all.
  • To add a port, firewall-cmd --add-port=80/tcp.
  • Instead of setting the ports manually, sometimes I can add one of the prebuilt services, such as nfs, firewall-cmd --add-service=nfs.
  • To make permanent changes, it is sufficient to add the --permanent flag.
  • To reload the firewall, firewall-cmd --reload. All runtime changes will be lost!
  • It is also possible panic mode, eveen though it is dangerous because it locks all the ports and kicks you out if you're remote: firewall-cmd --panic=on.
